# Client-Server Scalability: Distributed Message Ordering

**Authors:** Tam Vo, Brendan Nichols, Craig Beaubien

## About

This project investigates Client/Server Application architecture with an emphasis on large-scale scalable design and distributed algorithms. It utilizes **AWS Kinesis Data Streams** as a managed middleware solution to connect a high volume of client data with a cluster of backend servers. The primary goal is to explore how distributed systems can handle unpredictable traffic spikes while maintaining a high Quality of Service (QoS) for the end user.

The core application simulates distributed, high-frequency user telemetry data (mouse movements, vertical/horizontal scrolling patterns, and sub-second clickstreams) generated by an e-commerce website. We address the technical challenges of reliability and scalability in data ingestion and middleware processing. Specifically, because "exactly-once" delivery semantics introduce prohibitive coordination overhead in high-throughput environments, this system implements **"at-least-once" delivery combined with idempotent processing** and persistent state management.

## Key Research Focus Areas

*   **Hybrid & Predictive Autoscaling:** Investigating advanced Kubernetes autoscaling techniques beyond default Horizontal (HPA) and Vertical (VPA) scaling. We explore hybrid approaches that combine both methods, as well as predictive scaling using time-series forecasting (RNNs) to anticipate workload demands and make simultaneous scaling decisions.
*   **Latency-Aware Resource Allocation:** Monitoring stream-specific indicators such as **backpressure** (slow consumers forcing producers to slow down) and **lagging customers** (`MillisBehindLatest`) to intelligently adjust server provisions and avoid latency-induced oscillation.
*   **Idempotent State Management:** Preventing inaccurate data ingestion and handling the inevitable duplication of data inherent in low-latency, fault-tolerant stream processing architectures.
*   **Distributed Scalability:** Addressing Neuman's three dimensions of scale: numerical (handling increased load through replication), geographical (managing network latency via distributed middleware), and administrative (securely managing AWS cloud infrastructure policies).

## Directory Structure

The repository is structured around a multi-tier distributed architecture to ensure isolation of concerns and independent scalability of each component:

```text
├── client/          # E-commerce simulator generating high-frequency telemetry data
├── server/          # Backend nodes for Kinesis ingestion and idempotent processing
├── k8s/             # Kubernetes manifests (Deployments, Services, HPA/VPA configurations)
└── infrastructure/  # Cloud infrastructure as code (e.g., AWS Kinesis Data Streams)
```
